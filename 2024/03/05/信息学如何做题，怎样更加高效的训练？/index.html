    信息学如何做题，怎样更加高效的训练？

    发表于 2024-03-05

    提升代码水平

    Pro Tips — get them while they are free

    How to practice Competitive Programming [Um_nik version]

    一道题该怎么做，平时训练怎么做，做题步骤。

    看一道题

    不要跳着读，不要省时间粗略的读，捕获关键信息的能力，忽略题目的信息带来很惨重的信息。这段话没有出现数学语言不是关键的跳过。一字一句的读。他没有数学语言，陈述很关键的条件，比赛几个小时不差读题几分钟。都错题，之后的事情非常不好办。找到题目的关键信息，条件、数据范围特殊了然于心，看完提回顾一遍，这道题有什么特殊的地方。

    想题

    从性质、关键条件、性质去做这个题，有逻辑的寻找题目的突破口，而不是到处乱跳。相处一个题目在思维的地图找到那个点，不能随机，脑子中抛出很多想法， 逻辑的思考，不要任凭自己的直觉，准的还好，不准的不准。

    有逻辑地思考，从题目中提取关键信息，直接枚举做法

    不会不要慌，不要一下子想着做出题目，很多性质，一个个挖掘出来。尝试很多种思路、方向，位置很隐蔽，不符合第一第二直觉，不要觉得就应该用什么方式数据结构来做，尝试一个思路行不通不知道怎么做，立刻换一个思路，不要纠结在一个死胡同中。DP-》贪心，容斥，枚举，一个一个思路枚举，把脑海中的小技巧都便利一遍，找到合适的。
    尝试一个两个思路行不通，做不出来，就去看题解，这是大忌，
    一直尝试所有的思路直到黔驴技穷再看题解，独立思考的重要性，少看题解，多想
    自己做出来的题目和题目告诉你怎么做是不一样的，反复思考的重要性

    简化问题

    • 缩小数据范围，多次询问-单次询问
    • 图不会做，简化成树链，图-树-链。
    • 数多少个合法方案-简化成判断一个方案是否合法。
      尝试做一个简化，解决一个化简的问题，可能会发现适用原问题的关键性质。OI题，对着部分分，CF自己编部分分弱化条件，得到启发，实在不行，手摸一下小样例，博弈题，给出方法和过程，进来是否合法，通过模拟一下样例的过程更加了解问题的结构。不要只顾着机械的计算，NOIP考场上只会模拟样例，对着题意的条件算无用。再计算的同时停下来观察思考模拟的过程，思考博弈的性质，最优化长得非常像贪心或者DP之类的。
      也可以打表找规律，看题解之前前，千万不要再自己尝试所有办法之前不要看题解。
      题目错综复杂，可以在纸张电脑上记录下来，还是不会做，再重新读题，很常见，再题目没有读好的时候发生。

    想出做法

    想出来很兴奋，可能想直接写，保持冷静，把思路彻底捋清楚，think twice, code once。DP的状态，转移的设计需要把所有的细节想清楚再写。
    数据结构题，分好几个板块，版块怎么做怎么整合起来。对于某一个难写的地方，怎么好写。会一个题，做法麻烦稍微develop一下变成简单做法，心急没有想，直接写写不出来，调试不出来，常数太大，发展这个做法变得更好些常数更小。
    真实打比赛的时候，部分分，可能一档一档部分分的想，想完部分分的时候，定制比赛策略，策略可以灵活调整，怎么选择写哪一个部分分，预估写每一档部分分的时间和分值比对，在纸上罗列下来，冷静下来相信自己的直觉第一判断。分值比对，选定一个性价比最高当前的性价比最高最优策略，坚定的执行它，不要踌躇不前犹犹豫豫。CF没有部分分的比赛，到底写多少写哪个比较好。比赛技巧

    写代码

    每时每刻保持专注状态，写代码是机械化的过程，写着写着就变成刷着刷着变成自动挡，下意识的在哪里写，很容易写出很离谱的错误，无意识写代码-》专注的写代码，在比赛中太紧张压力太大，压力扰乱写代码，最好就是放松且专注的写代码。
    写代码非常常用的技巧：非常难写的部分先用暴力部分暴力写出来，写出一部分暴力之后只剩一部分暴力，拿样例测出来，改成想要实现的东西。信心大增，暴力-》正解，小暴力-高分暴力，暴力过样例-》，改成正解，获得启发，想出更好写的写法更优秀的实现方式。只会一个题的暴力。直接想正解，先写暴力。写着写着原来还可以这么些，没有想到的地方通往正解的一个突破口。

    调代码

    冷静下俩，不要绝望不要做无意义的输出调试。在你以为很容易写错的地方，其实出错的概率不大，在完全没有想到的地方出错，在容易出错的地方写代码实现的地方小心，比较容易的地方更容易进入刷牙的自动挡里，未曾涉足想到的会出错的地方。写一份代码停下来，瞪眼调试，尤其是专注度比较低的时候，先看一下有没有出错。实在调不出来，对拍，不要随机数据，要造一些更强的数据。对拍,直接随会挂分。

    做出来了

    不要直接关掉这个题，先反思，什么地方花了我太久，本应该很快想到但是很久。本来以为写代码很快但是像太久，看一下别人的代码。这个题值得记录用文字记录的地方很好的习惯。价值没那么高的不用记录，记录关键有用的信息。
